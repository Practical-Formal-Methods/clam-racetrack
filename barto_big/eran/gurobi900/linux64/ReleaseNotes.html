Release Notes for Version 9.0

In addition to performance enhancements, Gurobi 9.0 adds the following new features:

  • Non-Convex QP Support: We have added a new bilinear solver, which allows you to solve non-convex quadratic programming problems (i.e., QPs, QCPs, MIQPs, and MIQCPs). You will need to set the new NonConvex parameter to a non-default value to use this feature.
  • New Compute Server Capabilities: Gurobi Compute Server now includes a Cluster Manager, which adds many new features for administering your Compute Server cluster, including a Web User Interface, user management, job history, batch mode support, and more. See the Remote Services Reference Manual for details.
  • Batch Optimization in Compute Server: Compute Server now allows you to build a model on your client machine, submit it to your Compute Server, and then disconnect your client and retrieve results later (from your client or from another machine). See the Batch Optimization discussion for details. New attributes associated with batch mode: BatchErrorCode, BatchErrorMessage, BatchID, BatchStatus, VTag, CTag, QCTag, New parameters: JSONSolDetail,
  • Python Matrix API: We have extended our Python interface to support matrix-oriented modeling, using NumPy or SciPy matrices. Consult the documention for the new MVar object for details.
  • Piecewise-Linear (PWL) Constraints: We now support piecewise-linear constraints. For details, see the General Constraints discussion.
  • Function Constraints With Automatic PWL Translation: This release allows you to add nine basic nonlinear function constraints, including polynomial, exponential, logarithmic and trigonometric functions. We will perform an automatical piecewise-linear approximiation of these functions. For details see the General Constraints discussion. New attributes associated with function constraints: FuncPieces, FuncPieceError, FuncPieceLength, FuncPieceRatio. New parameters: FuncMaxVal, FuncPieces, FuncPieceError, FuncPieceLength, FuncPieceRatio.
  • MIP Scenario Analysis: This new feature allows you to describe multiple sets of changes to a model (scenarios), and computes the impact of these changes on the solution. See the new Multiple Scenarios section for details. New attributes associated with scenarios: ScenNLB, ScenNUB, ScenNObj, ScenNRHS, ScenNName, ScenNX, ScenNObjBound, ScenNObjVal, NumScenarios. New parameters: ScenarioNumber,
  • Intermediate Solution Files: We now allow you to write solutions found during the MIP solution process to files as soon as they are found, using the new SolFiles parameter.
  • Lazy Constraint Callback: We now support lazy constraint callbacks for Compute Server.
  • Indices for Variables and Constraints: You can now query the index of a variable or a constraint in the underlying constraint matrix for the OO APIs (Python, C++, Java and .NET). Look for the new index method on these objects (e.g. Var.index in Python).
  • A New Model Attribute File: The Gurobi attribute file format consolidates attribute settings for a number of user-specified attributes. For details see ATTR format section.
  • Python Version for Interactive Shell: The included Gurobi Interactive Shell has been updated from Python 2.7 to 3.7.
  • New parameters: In addition to the attributes and parameters directly tied to the new features above, there are three additional new parameters:
    • BQPCuts New cutting plane type.
    • RelaxLiftCuts New cutting plane type.
    • RLTCuts New cutting plane type.

Code Conversion from Version 8.1

Programs that were originally written for version 8.1 of the Gurobi Optimizer generally won't require any code changes to use version 9.0. If you'd like to use the new features, some code changes are required.

One minor exception is MATLAB or R programs that retrieve quadratic constraints from a model. The associated matrices are now returned in a different format. Refer to the MATLAB documentation or R documentation for more information.

Another minor exception is the GRBfixedmodel routine in C, which is now deprecated and has been replaced by the new GRBfixmodel routine.
